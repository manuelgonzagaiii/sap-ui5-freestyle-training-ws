# Open-Source Backend with SAP CAP

> **Track:** JavaScript  ·  **Phase D · Data & Backend**
> **Status:** Skeleton — stages to be populated.

This is a **guided-project (framework) lesson**: your app code carries forward
from one stage to the next, and from the previous lesson into this one. This
overview page is the roadmap for the lesson; each numbered item below will
become a hands-on stage.

## What you'll learn
- SAP Cloud Application Programming model (CAP) — free, open source
- CDS data model for Products, SalesOrders, Customers (+ associations)
- Exposing an OData V4 service and seeding SQLite sample data
- Running `cds watch` locally with no paid systems

## Planned stages
1. Scaffold a CAP project (db/ schema.cds, srv/ service.cds)
2. Model entities + associations and add CSV seed data
3. Run cds watch with sqlite and inspect $metadata
4. Add a draft-enabled service and a bound action
5. Point the UI5 app's dataSource at the CAP OData V4 URL

## SAPUI5 features covered
- `SAP CAP (cds)`
- `OData V4 service`
- `SQLite`
- `CSV seed data`
- `draft @odata.draft.enabled`

## Backend for this lesson
**SAP CAP + SQLite (OData V4)**
