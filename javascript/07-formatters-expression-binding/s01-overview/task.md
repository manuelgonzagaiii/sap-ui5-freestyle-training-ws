# Formatters, Types & Expression Binding

> **Track:** JavaScript  ·  **Phase B · UI Composition**
> **Status:** Skeleton — stages to be populated.

This is a **guided-project (framework) lesson**: your app code carries forward
from one stage to the next, and from the previous lesson into this one. This
overview page is the roadmap for the lesson; each numbered item below will
become a hands-on stage.

## What you'll learn
- Formatter functions and where to register them
- Expression binding for simple inline logic
- Composite binding (parts) and calculated fields
- Custom sap.ui.model.SimpleType data types with formatValue/parseValue/validateValue

## Planned stages
1. Add a formatter.js module and format a status into a state
2. Use expression binding to toggle visibility/enabled
3. Combine first+last name with composite binding parts
4. Create a custom data type for a SKU/price
5. Bind ObjectStatus state via a formatter

## SAPUI5 features covered
- `formatter functions`
- `expression binding`
- `composite binding`
- `custom SimpleType`
