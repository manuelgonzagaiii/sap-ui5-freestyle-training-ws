# Accessibility (a11y)

> **Track:** TypeScript  ·  **Phase E · Advanced & Rare**
> **Status:** Skeleton — stages to be populated.

This is a **guided-project (framework) lesson**: your app code carries forward
from one stage to the next, and from the previous lesson into this one. This
overview page is the roadmap for the lesson; each numbered item below will
become a hands-on stage.

## What you'll learn
- ARIA roles/labels and the UI5 accessibility model
- Keyboard handling and F6 fast navigation groups
- Screen-reader semantics, ariaLabelledBy/describedBy
- High-contrast themes and accessibility testing

## Planned stages
1. Add labels + ariaLabelledBy across the forms
2. Define F6 fast-navigation groups
3. Verify keyboard operability for tables/dialogs
4. Test with a high-contrast theme
5. Audit with the accessibility checks

## SAPUI5 features covered
- `ARIA support`
- `F6 fast navigation`
- `ariaLabelledBy/describedBy`
- `high-contrast (hcb)`

## Backend for this lesson
**SAP CAP + SQLite (OData V4)**
