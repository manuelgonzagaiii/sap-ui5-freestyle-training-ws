# Routing & Navigation

> **Track:** TypeScript  ·  **Phase B · UI Composition**
> **Status:** Skeleton — stages to be populated.

This is a **guided-project (framework) lesson**: your app code carries forward
from one stage to the next, and from the previous lesson into this one. This
overview page is the roadmap for the lesson; each numbered item below will
become a hands-on stage.

## What you'll learn
- Configuring routing in the manifest: routes, targets, patterns
- Pattern + query parameters and pattern matching events
- Nested routes and multiple targets
- Programmatic navTo, history, browser back and not-found handling

## Planned stages
1. Declare a router with a list route and an object route (pattern {id})
2. Navigate list→detail with navTo and bind by route parameter
3. Handle routePatternMatched to bindElement on the detail page
4. Add a notFound target and a 404 view
5. Wire browser back via History + navBack

## SAPUI5 features covered
- `sap.ui.core.routing.Router`
- `routes/targets/patterns`
- `route parameters`
- `History/notFound`
