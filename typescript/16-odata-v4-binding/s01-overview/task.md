# OData V4 Binding

> **Track:** TypeScript  ·  **Phase D · Data & Backend**
> **Status:** Skeleton — stages to be populated.

This is a **guided-project (framework) lesson**: your app code carries forward
from one stage to the next, and from the previous lesson into this one. This
overview page is the roadmap for the lesson; each numbered item below will
become a hands-on stage.

## What you'll learn
- sap.ui.model.odata.v4.ODataModel and its binding model
- Context binding, list binding, $expand/$select/$count
- Parameter contexts, operation bindings (bound/unbound actions & functions)
- Batch groups, $auto and refresh

## Planned stages
1. Configure the V4 model + dataSource from the CAP service
2. Bind a table with $select/$expand and $count
3. Bind a detail with a context binding and relative bindings
4. Invoke a bound action and a function import
5. Tune update/refresh groups ($auto vs deferred)

## SAPUI5 features covered
- `odata.v4.ODataModel`
- `context/list binding`
- `$expand/$select/$count`
- `operation bindings`

## Backend for this lesson
**SAP CAP + SQLite (OData V4)**
