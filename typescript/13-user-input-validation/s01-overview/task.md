# Forms, Input & Validation

> **Track:** TypeScript  ·  **Phase C · Interaction**
> **Status:** Skeleton — stages to be populated.

This is a **guided-project (framework) lesson**: your app code carries forward
from one stage to the next, and from the previous lesson into this one. This
overview page is the roadmap for the lesson; each numbered item below will
become a hands-on stage.

## What you'll learn
- SimpleForm and Form with responsive layouts
- Input controls and sap.ui.model types with constraints
- Client validation, handleValidation and the Message Manager binding
- Field groups, required fields and value state

## Planned stages
1. Build an edit form with SimpleForm + ResponsiveGridLayout
2. Apply typed bindings (Integer/Float/Date/String with constraints)
3. Enable handleValidation in the manifest and show value states
4. Bind messages to the form and open a MessagePopover on error
5. Group fields and validate on submit

## SAPUI5 features covered
- `sap.ui.layout.form.SimpleForm/Form`
- `model types + constraints`
- `handleValidation`
- `valueState`
