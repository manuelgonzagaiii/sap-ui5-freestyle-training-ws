# Controls & Libraries

> **Track:** TypeScript  ·  **Phase A · Foundations**
> **Status:** Skeleton — stages to be populated.

This is a **guided-project (framework) lesson**: your app code carries forward
from one stage to the next, and from the previous lesson into this one. This
overview page is the roadmap for the lesson; each numbered item below will
become a hands-on stage.

## What you'll learn
- The control metadata model: properties, aggregations, associations, events
- Touring sap.m (responsive) vs sap.ui.core base controls
- App scaffolding controls: App, Page, Bar, Toolbar, OverflowToolbar
- Finding controls in the API Reference and the Demo Kit samples

## Planned stages
1. Build a Page with a custom header Bar and footer Toolbar
2. Add common sap.m controls: Input, Select, CheckBox, RadioButton, Slider
3. Use aggregations vs associations (e.g. label 'labelFor')
4. Handle control events and read values
5. Group actions in an OverflowToolbar with priorities

## SAPUI5 features covered
- `control metadata`
- `sap.m core controls`
- `App/Page/Bar/Toolbar`
- `OverflowToolbar`
