# Performance & Best Practices

> **Track:** TypeScript  ·  **Phase F · Quality & Delivery**
> **Status:** Skeleton — stages to be populated.

This is a **guided-project (framework) lesson**: your app code carries forward
from one stage to the next, and from the previous lesson into this one. This
overview page is the roadmap for the lesson; each numbered item below will
become a hands-on stage.

## What you'll learn
- Async everywhere: async bootstrap, async manifest, lazy loading
- Dependency declaration, asyncHints and Component-preload
- Model size limits, growing/threshold and binding performance
- Diagnostics, the Support Assistant and the UI5 Inspector

## Planned stages
1. Audit and enforce async loading end-to-end
2. Declare libs/components for preload and asyncHints
3. Tune list growing/threshold and sizeLimit
4. Run the Support Assistant and fix findings
5. Profile with the UI5 Diagnostics + Inspector

## SAPUI5 features covered
- `async manifest/bootstrap`
- `Component-preload / asyncHints`
- `model sizeLimit`
- `Support Assistant`

## Backend for this lesson
**SAP CAP + SQLite (OData V4)**
