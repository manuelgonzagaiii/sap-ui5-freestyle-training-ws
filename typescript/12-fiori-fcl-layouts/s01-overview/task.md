# Fiori Layouts (FCL, DynamicPage, ObjectPage)

> **Track:** TypeScript  ·  **Phase C · Interaction**
> **Status:** Skeleton — stages to be populated.

This is a **guided-project (framework) lesson**: your app code carries forward
from one stage to the next, and from the previous lesson into this one. This
overview page is the roadmap for the lesson; each numbered item below will
become a hands-on stage.

## What you'll learn
- sap.f.FlexibleColumnLayout and the list-detail-detail pattern
- sap.f.DynamicPage and the semantic page header
- sap.uxap.ObjectPageLayout with anchored sections
- Layout state, arrow navigation and routing integration

## Planned stages
1. Introduce a FlexibleColumnLayout with begin/mid columns
2. Drive FCL layout from the router (OneColumn → TwoColumnsMidExpanded)
3. Build a DynamicPage detail header with snapping
4. Render an ObjectPageLayout with multiple sections
5. Handle column resize + close in the FCL

## SAPUI5 features covered
- `sap.f.FlexibleColumnLayout`
- `sap.f.DynamicPage`
- `sap.uxap.ObjectPageLayout`
- `semantic page`
